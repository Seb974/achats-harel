Section 0

Présentation générale d'Odoo

Comprendre l'outil qui gère les opérations d'AH CHOU

🏢 Qu'est-ce qu'Odoo ?

Odoo est un logiciel de gestion d'entreprise (ERP) tout-en-un. Il regroupe dans une seule interface tous les outils nécessaires pour gérer les achats, les stocks, les ventes et la comptabilité.

Son fonctionnement est modulaire : chaque fonction métier est un module indépendant, mais tous communiquent entre eux automatiquement. Quand vous réceptionnez une commande d'achat, le stock se met à jour tout seul.

💡

Odoo est accessible depuis n'importe quel navigateur web — pas besoin d'installer de logiciel sur votre ordinateur.

📦 Modules utilisés chez AH CHOU

1

Créer un achat dans l'application

Depuis l'application web, créez un nouvel achat :

  • Sélectionnez le fournisseur dans la liste déroulante
  • Ajoutez les produits souhaités depuis le catalogue (299 produits surgelés disponibles)
  • Indiquez les quantités pour chaque produit
  • Vérifiez les prix unitaires — ils sont pré-remplis depuis Odoo mais modifiables
💡

L'achat reste en statut Brouillon tant que vous ne l'envoyez pas. Vous pouvez le modifier librement.

2

Envoyer l'achat → Commande créée dans Odoo

Lorsque vous cliquez sur « Envoyer », l'application :

  • Crée automatiquement un bon de commande (Purchase Order) dans Odoo
  • Le confirme pour déclencher la chaîne logistique
  • Déplace la carte dans la colonne ENVOYÉ
Liste des bons de commande dans le module Achats d'Odoo
ℹ️

À partir de ce moment, le bon de commande est visible dans Odoo → Achats → Bons de commande. Le champ x_statut_transit suit la progression.

3

Suivre le transit maritime via le Kanban

La marchandise voyage de l'étranger vers La Réunion. Suivez sa progression en glissant la carte d'une colonne à la suivante :

  • ENVOYÉ → EN MER : la marchandise a quitté le port d'origine
  • EN MER → AU PORT : le navire est arrivé au port de La Réunion
  • AU PORT → DOUANE : la marchandise est en cours de dédouanement
💡

Chaque déplacement crée un transfert interne dans Odoo (module Stock). La marchandise passe d'un emplacement de transit au suivant.

4

Dédouanement → A réceptionner

Lorsque vous glissez la carte de DOUANE vers A RÉCEPTIONNER, une boîte de dialogue s'affiche pour :

  • Confirmer le prix de revient final (PR) pour chaque ligne de produit
  • Ce prix intègre les frais de transport, douane et autres coûts
  • Il est essentiel pour le calcul correct des marges
⚠️

Attention : vérifiez soigneusement le prix de revient avant de valider. Ce prix sera utilisé pour calculer les 7 listes de prix (CATALOGUE, HYPER, SUPER, GROSSISTE, PLATEFORME, CHR, PARTENAIRE).

5

Réception dans Odoo

Une fois dans la colonne A RÉCEPTIONNER, un bouton « Réceptionner dans Odoo » apparaît sur la carte :

  • Cliquez dessus pour ouvrir directement le formulaire de réception (picking) dans Odoo
  • Dans Odoo, pour chaque ligne de produit :
    • Saisissez le numéro de lot
    • Indiquez la date limite de consommation (DLC)
    • Confirmez la quantité reçue
  • Validez la réception dans Odoo
Formulaire d'un bon de commande dans Odoo (détail des lignes produits)
🚨

Obligatoire : chaque produit surgelé doit avoir un numéro de lot et une DLC. Sans ces informations, la réception ne peut pas être validée dans Odoo (traçabilité réglementaire).

💡

L'application vérifie automatiquement toutes les 60 secondes si la réception a été finalisée dans Odoo (auto-polling).

6

Achat marqué « Reçu »

Dès qu'Odoo confirme la réception complète :

  • La carte passe automatiquement dans la colonne REÇU
  • Le stock est mis à jour dans Odoo avec les quantités, lots et DLC saisis
  • Les 7 listes de prix sont recalculées à partir du prix de revient confirmé
  • La marchandise est désormais disponible à la vente
💡

Vous pouvez vérifier le stock mis à jour dans Odoo → Stock/Inventaire → Produits.

✅ Récapitulatif du flux achats

App (création) → Brouillon → Envoyé (BC Odoo créé) → En Mer → Au Port → Douane → A Réceptionner (vérification PR) → Odoo (réception lots/DLC) → Reçu (stock à jour)

Tout le suivi logistique se fait dans l'application. Seule la réception finale nécessite d'intervenir directement dans Odoo.

2. Réception & Gestion du Stock

Réceptionner les marchandises, consulter le stock, gérer les emplacements et les dates de péremption.

2.1

Réceptionner une commande

Lorsque la marchandise arrive à l'entrepôt, vous devez enregistrer la réception dans Odoo pour mettre à jour le stock et assurer la traçabilité par lot.

1

Accéder aux réceptions

Deux possibilités :

  • Depuis Odoo : allez dans Inventaire → Opérations → Réceptions.
  • Depuis l'application AH CHOU : cliquez sur le bouton « Réceptionner dans Odoo » qui ouvre directement la liste des réceptions en attente.
Vue d'ensemble du module Inventaire — réceptions, transferts internes, livraisons
2

Ouvrir le bon de réception

Repérez le bon de réception correspondant à votre commande. Il porte un numéro de type WH/IN/00045. Cliquez dessus pour l'ouvrir.

Vous verrez la liste des produits commandés avec les quantités attendues.

3

Renseigner les informations de chaque produit

Pour chaque ligne de produit, complétez les champs suivants :

  • Numéro de lot — Saisissez ou scannez le numéro de lot (ex : LOT-2026-001). Si le lot n'existe pas encore, Odoo le créera automatiquement.
  • Date de péremption (DLC) — Indiquez la date limite de consommation figurant sur l'emballage. Cette date est obligatoire pour tous les produits surgelés.
  • Quantité reçue — Saisissez la quantité effectivement reçue. Si elle diffère de la quantité commandée, Odoo créera un reliquat automatiquement.
  • Emplacement de destination — Choisissez le congélateur ou la zone de stockage (Congélateur 1, 2, 3, Réfrigéré, etc.).
Astuce : Scanner les codes-barres Si vous disposez d'un lecteur de codes-barres, vous pouvez scanner le numéro de lot directement. Odoo remplira automatiquement le champ correspondant.
Attention : Date de péremption obligatoire Tous les 299 produits du catalogue AH CHOU sont configurés avec le suivi par lot et la date de péremption obligatoire. Vous ne pourrez pas valider la réception si la DLC n'est pas renseignée.
4

Valider la réception

Une fois toutes les lignes complétées, cliquez sur le bouton « Valider » en haut du formulaire. Le stock sera immédiatement mis à jour dans les emplacements choisis.

Si certaines quantités sont inférieures à la commande, Odoo vous proposera de créer un reliquat (backorder) pour recevoir le reste ultérieurement.

Bon à savoir Après validation, le statut du bon de réception passe à « Fait ». Vous pouvez retrouver l'historique de toutes les réceptions validées en filtrant par statut dans la liste des réceptions.

Qu'est-ce que la stratégie FEFO ?

FEFO signifie « First Expired, First Out » — en français : « Premier Périmé, Premier Sorti ».

Concrètement, lorsque vous préparez une commande, Odoo vous proposera automatiquement les lots dont la date de péremption est la plus proche. Cela garantit :

  • La rotation optimale des stocks
  • La réduction des pertes liées aux produits périmés
  • Le respect des normes sanitaires pour les produits surgelés
Fonctionnement automatique Vous n'avez rien à configurer : les emplacements de stockage (congélateurs et réfrigéré) utilisent déjà la stratégie FEFO. Odoo sélectionnera les bons lots pour vous lors de la préparation des commandes.
2.4

Alertes DLC (dates de péremption)

La gestion des dates de péremption est essentielle pour les produits surgelés. Odoo vous aide à surveiller les DLC grâce à plusieurs outils.

Vérifier les produits proches de la péremption

1

Accéder à la liste des lots

Allez dans Inventaire → Produits → Lots/Numéros de série.

2

Filtrer par date d'alerte

Utilisez le filtre « Date d'alerte » pour afficher uniquement les lots dont la date d'alerte est dépassée ou proche. Vous pouvez aussi trier la colonne « Date de péremption » par ordre croissant.

3

Agir sur les produits concernés

Pour chaque lot proche de la péremption :

  • Le vendre en priorité (la stratégie FEFO le fait automatiquement)
  • Le transférer en zone Quarantaine si la DLC est dépassée
  • Le détruire via un ajustement d'inventaire

2.5

Transferts internes

Un transfert interne permet de déplacer du stock d'un emplacement à un autre au sein de l'entrepôt, sans changer de propriétaire.

Quand faire un transfert interne ?

  • Déplacer des produits d'un congélateur vers la zone Picking pour préparer une commande
  • Réorganiser le stock entre congélateurs (ex : Congélateur 1 → Congélateur 2)
  • Mettre un produit en Quarantaine (produit endommagé, périmé ou non conforme)
  • Transférer du stock vers le site Logistisud (stock déporté)
  • Ramener du stock depuis Logistisud vers l'entrepôt principal

1

Créer un transfert interne

Allez dans Inventaire → Opérations → Transferts internes, puis cliquez sur « Nouveau ».

2

Renseignez :

  • Emplacement source — d'où partent les produits (ex : WH/Stock/CONGELATEUR-1)
  • Emplacement de destination — où arrivent les produits (ex : WH/Stock/PICKING)
3

Ajouter les produits à transférer

Cliquez sur « Ajouter une ligne » et sélectionnez le produit, le numéro de lot et la quantité à déplacer.

4

Astuce : Transfert rapide Si vous êtes sur la fiche d'un produit et que vous voyez son stock dans un emplacement, vous pouvez aussi utiliser le menu « Déplacer » pour initier un transfert plus rapidement.
Attention : Lot obligatoire Tous les produits AH CHOU sont suivis par lot. Lors d'un transfert interne, vous devez toujours sélectionner le numéro de lot à transférer.

⚠️

Livraisons partielles (Backorders)
Si le stock disponible ne couvre pas la totalité de la commande, Odoo propose de créer un reliquat. Un second BL sera généré automatiquement pour les quantités restantes, livrable lors de la prochaine tournée.

ℹ️

Tournées de livraison
Les livraisons sont organisées par zone (x_zone_livraison), jour (x_jour_livraison) et tournée (x_tournee). L'entrepôt effectue en moyenne 2 à 3 tournées par jour.

🚨

Lot obligatoire
Tous les 299 produits surgelés AH CHOU sont suivis par lot. Lors de la préparation (Pick), le lot doit être renseigné. Odoo le propose automatiquement via FEFO, mais le préparateur doit valider la sélection.

Le mode de facturation est défini sur la fiche client par le champ x_groupe_facturation. Il est configuré lors de la création du client et peut être modifié à tout moment.

3.4 Retours et Avoirs

↩️
Gérer un retour produit
  1. Depuis le bon de livraison validé, cliquez sur le bouton Retour.
  2. Sélectionnez les produits retournés et indiquez le motif : DLC dépassée, casse, erreur.
  3. Validez le retour. Le stock est transféré vers l'emplacement QUARANTAINE.
  4. Créez un avoir (note de crédit) depuis la facture d'origine → Ajouter une note de crédit.
🚨

Important — Emplacement QUARANTAINE
Les produits retournés ne réintègrent jamais directement le stock vendable. Ils sont placés en QUARANTAINE pour inspection. Seul le responsable logistique (Marc Hareng) peut décider de les reclasser ou de les détruire.

3.5 Suivi des paiements

💰
Suivre les encaissements clients
  1. Accédez à Comptabilité → Rapports → Balance âgée clients pour visualiser les factures en attente de règlement.
  2. Identifiez les clients dont l'encours dépasse le seuil d'alerte (credit_limit).
  3. Pour enregistrer un paiement, ouvrez la facture et cliquez sur Enregistrer un paiement. Sélectionnez le journal de paiement (banque, espèces, chèque) et validez.
⚠️

Encours client
Le champ credit_limit est défini sur chaque fiche client. Lorsqu'un commercial (Sarah Dine, Sam Oussah) crée un devis et que l'encours dépasse la limite, un message d'alerte s'affiche. La direction (Paul Pieuvre) peut alors décider de bloquer ou d'autoriser la commande.

🔀 Variantes produit — 1 Gencod = plusieurs fiches

Certains produits AH CHOU partagent le même code Gencod (EAN) mais sont déclinés par origine géographique. C'est le cas de 23 groupes de produits.

Fonctionnellement

Un même produit (ex: Carri de daurade PNE 360g) peut provenir du Vietnam ou d'Indonésie. Le Gencod est identique car c'est le même produit commercial, mais Odoo a besoin de fiches distinctes pour gérer les origines, coûts et traçabilité séparément.

Techniquement dans Odoo
  • L'attribut de variante « Origine » est configuré sur le produit template
  • Chaque origine crée un product.product distinct (variante)
  • Le champ x_gencod conserve le code EAN commun pour référence commerciale
  • Le champ barcode est unique par variante (seule la fiche principale le porte)
  • Chaque variante a son propre stock, lot et prix de revient

203156Carri de daurade Vietnam PNE 360G3456470203156(vide)

Même Gencod 3456470203156, mais 2 fiches produit avec origines et coûts différents.
