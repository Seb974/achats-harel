## Conseils pour les captures d'écran

### Qualité
- Utiliser une résolution suffisante (1920x1080 minimum)
- Capturer la fenêtre entière ou la zone pertinente
- S'assurer que le texte est lisible
- Éviter les captures floues

### Contenu
- Masquer les données sensibles si nécessaire
- Afficher les éléments importants (menu, champs, valeurs)
- Inclure les titres de section pour contexte
- Montrer les messages de succès/erreur si pertinents

### Format
- **Format recommandé:** PNG (meilleure qualité pour captures d'écran)
- **Format alternatif:** JPG (si taille de fichier importante)
- **À éviter:** GIF (sauf pour animations)

### Outils de capture
- **macOS:** Cmd + Shift + 4 (sélection) ou Cmd + Shift + 3 (écran complet)
- **Windows:** Win + Shift + S (Outil Capture d'écran)
- **Linux:** gnome-screenshot ou Spectacle
- **Navigateur:** Extensions de capture d'écran (Awesome Screenshot, etc.)

## Stockage et sauvegarde

### Emplacement
- **Local:** `docs/integration_odoo_ah_chou/BLOC1_SCREENSHOTS/`
- **Sauvegarde recommandée:** Stockage cloud (Google Drive, Dropbox, etc.)
- **Versioning:** Git (si taille raisonnable) ou Git LFS

### Taille maximale
- Par fichier : < 5 MB
- Total du dossier : < 100 MB (recommandé)

Si le dossier devient trop volumineux :
- Compresser les images (TinyPNG, ImageOptim, etc.)
- Utiliser Git LFS pour le versioning
- Créer une archive ZIP pour backup

- Les captures d'écran sont des documents de référence
