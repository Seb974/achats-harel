Bloc 3 : Tarification

Listes de Prix

Système de tarification par liste avec calcul automatique basé sur le prix d'achat :

CodeNomCalculCible
00Tarif PublicPrix d'achat + 35%Particuliers, petits commerces
90Tarif GrossistePrix d'achat + 25%Grossistes
91Tarif GMSPrix d'achat + 20%Grandes surfaces
92Tarif RestaurationPrix d'achat + 22%Restaurants, hôtels
93Tarif ExportPrix d'achat + 18%Clients export
Mécanisme de Calcul

Formule : Prix de vente HT = Prix d'achat × (1 + Marge%)

Le prix d'achat utilisé est le Prix d'achat standard (champ standard_price) mis à jour à chaque réception.

Exemple de calcul

ProduitPrix achatTarif 00 (+35%)Tarif 91 (+20%)
Crevettes 16/208,50 €11,48 €10,20 €
Bichique Premium12,00 €16,20 €14,40 €

Stratégie FEFO (First Expired, First Out)

FEFO activé sur toutes les zones de stockage

Les produits avec la date d'expiration la plus proche sont prélevés en priorité.

Fonctionnement

  1. Réception : Création d'un lot avec date d'expiration (DLC/DLUO)
  2. Stockage : Rangement dans l'emplacement approprié
  3. Picking : Odoo propose automatiquement le lot qui expire en premier
  4. Alertes : Notification pour les produits proches de l'expiration

Prérequis : Le module "Dates d'expiration des produits" doit être activé et la traçabilité par lots configurée sur chaque produit.

Gestion des Variantes (Codes-barres)

Certains produits partagent le même code-barre car ils représentent le même article avec des origines différentes.

Solution mise en place : Utilisation de l'attribut "Origine" pour créer des variantes de produit.

Exemple

Produit parentVariantes
Carri de thon 400g • Origine Inde
• Origine Vietnam
• Origine Indonésie
Escalope de marlin 400g • Origine Inde
• Origine Vietnam
• Origine Indonésie

Chaque variante peut avoir son propre code-barre, prix et niveau de stock.
